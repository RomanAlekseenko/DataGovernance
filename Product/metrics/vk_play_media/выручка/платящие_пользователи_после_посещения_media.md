# Платящие пользователи после посещения Media

## Бизнес-определение
Метрика показывает количество уникальных пользователей, которые после посещения VK Play Media совершили покупку в другом домене экосистемы VK Play по session-level или assisted-сценарию attribution.

## Зачем используется
* Для оценки общего числа монетизирующихся пользователей после контакта с Media beyond click-confirmed transitions.
* Для сравнения узкого direct-контура и более широкого post-visit revenue-слоя.
* Для расчета ratio-метрик revenue-контура второго слоя.

## Условия расчета
Метрика считается как количество уникальных пользователей, у которых после посещения Media произошла хотя бы одна атрибутированная покупка по `session-level` или `assisted` сценарию.

Зафиксированное правило:
* если одна и та же транзакция одновременно подходит под несколько сценариев, применяется приоритет `direct > session-level > assisted`;
* одна транзакция может быть отнесена только к одному attribution-сценарию;
* окно revenue-attribution составляет `3` дня.

### Включаются
* Пользователи с хотя бы одной валидной атрибутированной покупкой после посещения Media.
* Покупки в других доменах экосистемы VK Play, атрибутированные по `session-level` или `assisted`.

### Не включаются
* Пользователи без подтвержденной атрибутированной покупки.
* Пользователи, чьи покупки уже попали в direct-метрику, если та же транзакция не может быть продублирована в post-visit контуре.

## Как интерпретировать изменения
Рост:
Рост означает увеличение числа пользователей, которые монетизируются после контакта с Media.

Падение:
Падение означает снижение post-visit monetization-эффекта Media на уровне пользователей.

## Ограничения и предостережения
* Метрика зависит от качества user matching между Media и транзакциями.
* Один пользователь с несколькими покупками считается один раз.
* Без фиксированного правила транзакционных статусов и возвратов значение может быть нестабильным.

## Минимальные проверки качества
* Для атрибутированных покупок доступен стабильный user key.
* Значение платящих пользователей не превышает число пользователей, попавших в post-visit revenue-контур.
* Метрика сверяется с `Выручка после посещения Media` и `Assisted revenue after Media`.
