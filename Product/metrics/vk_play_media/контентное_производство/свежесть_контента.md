# Свежесть контента

## Бизнес-определение
Метрика показывает долю потребления VK Play Media, приходящуюся на материалы, опубликованные в пределах согласованного окна свежести.

## Домен
VK Play Media

## Сквозная метрика или нет
Нет

## Владелец
Руководитель отдела аналитики и данных.

## Зачем используется
* Для оценки того, насколько потребление Media опирается на свежий контент.
* Для управления балансом между новым и архивным контентом.
* Для контроля актуальности редакционного потока.

## Источник истины
Составная модель на базе CMS и данных просмотров материалов.

## Гранулярность
* по дням;
* по неделям;
* по материалам;
* по категориям.

## Частота обновления
Ежедневно до 10:00.

## Условия расчета
Метрика считается как доля просмотров в отчетном периоде, пришедшихся на материалы, возраст которых на момент просмотра не превышал согласованное окно свежести.

Дата публикации материала не ограничивает его участие в расчете сама по себе. В расчет попадает любой материал, который получил просмотры в отчетном периоде и для которого можно определить возраст материала на момент просмотра.

Зафиксированное окно свежести - `7` календарных дней с даты публикации.
Обновление материала без повторной публикации не меняет дату публикации для расчета свежести.

### Включаются
* Просмотры материалов, для которых известна дата первой публикации.

### Не включаются
* Просмотры материалов без корректной даты публикации.
* Черновики и служебные материалы.

## Как интерпретировать изменения
Рост:
Рост означает, что аудитория активнее потребляет свежий контент и редакционный поток лучше поддерживает текущий интерес.

Падение:
Падение означает, что большая доля потребления уходит в архив или что новый контент недостаточно быстро набирает спрос.

## Ограничения и предостережения
* Значение полностью зависит от выбранного окна свежести.
* Для evergreen-контента низкая свежесть не обязательно является проблемой.
* Сравнение периодов возможно только при неизменном правиле публикационной даты и окна свежести.
* Обновленные материалы без повторной публикации продолжают учитываться по дате первой публикации, даже если содержательно были существенно переработаны.
* Без подтвержденного поля даты первой публикации в CMS метрика не может считаться воспроизводимо.

## Минимальные проверки качества
* Для материалов заполнена дата первой публикации.
* Окно свежести зафиксировано в определении метрики: `7` календарных дней.
* Просмотры по материалам без даты публикации выделяются отдельно.
