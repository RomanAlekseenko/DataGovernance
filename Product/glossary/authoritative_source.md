# Авторитетный источник данных

## Краткое определение
Авторитетный источник данных - это таблица, витрина, модель или система, которая считается основной и приоритетной для расчета конкретной метрики, сущности или атрибута.

## Полное бизнес-определение
Термин `authoritative source` используется для снятия конфликтов между несколькими возможными источниками.
Если для одного и того же показателя существует несколько таблиц или отчетов, authoritative source должен явно определять, какой из них считается основным для принятия решений и почему.

## Где используется
* доменные документы;
* метрики;
* описания датасетов и витрин;
* quality checks;
* change management и impact assessment.

## Что входит в термин
* источник, выбранный как основной для конкретного сценария использования;
* источник, по которому можно воспроизвести расчет;
* источник, у которого определены owner, grain и SLA, либо явно зафиксировано их отсутствие как `TBD:`.

## Что не входит в термин
* любой удобный источник для разового анализа;
* промежуточная выгрузка без статуса основного источника;
* конкурирующий источник без правила приоритета.

## Правила интерпретации
* Authoritative source должен определяться для каждой критичной метрики и сущности.
* Один и тот же источник не обязан быть authoritative для всех сценариев сразу.
* Если authoritative source не определен, определение метрики или сущности считается незавершенным.
