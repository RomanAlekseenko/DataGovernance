# Glossary

## Назначение
Этот каталог фиксирует кросс-доменные термины, которые должны использоваться единообразно в доменных описаниях, метриках, датасетах и дашбордах.

Glossary нужен для того, чтобы:
* снижать риск разных трактовок одного и того же термина;
* не дублировать полные определения в каждом домене;
* задавать базовый semantic layer для метрик и отчетности;
* упрощать согласование между бизнесом, аналитиками, BI и data engineering.

## Когда термин нужно выносить в glossary
Термин стоит выносить в glossary, если он:
* используется более чем в одном домене;
* встречается в нескольких метриках или дашбордах;
* влияет на трактовку бизнес-результата;
* регулярно вызывает вопросы или спорные интерпретации;
* требует единого стандарта для source of truth, ownership или quality checks.

## Правила ведения
* Один файл соответствует одному термину.
* Название файла должно быть коротким, в `snake_case`, на русском или в устойчивой рабочей форме.
* Если термин еще не согласован полностью, в документе должны быть `TBD:` и `Открытый вопрос:`.
* Если термин используется в домене, желательно давать ссылку на glossary из доменного README или из метрики.
* Если изменение термина меняет смысл метрик или отчетов, это должно рассматриваться как потенциальный `breaking change`.
